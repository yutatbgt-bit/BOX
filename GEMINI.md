# Python開発環境構築の方針 (Ubuntu/WSL + uv)


＃UbuntuでWSL接続時のLinux環境での開発時は以下のルールを厳守
## 1. 基本アーキテクチャ
- **ツール:** `uv` を全面的に採用し、Pythonのインストール、パッケージ管理、仮想環境構築を一元管理する。
- **Python管理:** `uv python install` を使用し、システムPythonには一切変更を加えない。
- **仮想環境:** プロジェクト直下の `.venv` ディレクトリに構築する（In-project virtualenvs）。
- **依存関係管理:** `pyproject.toml` および `uv.lock` を厳格に使用し、決定論的なビルド（Deterministic build）を保証する。

## 2. 堅牢なセットアップ手順（新規プロジェクト時）
新規プロジェクト作成の指示があった場合は、以下のフローを提案・実行すること：
1. **初期化:** `uv init --app` または `uv init --lib`
2. **Pythonバージョン固定:** `.python-version` ファイルの作成
3. **設定の堅牢化:** `uv.toml` を作成し、`dev-dependencies` や `index-url` 等を明示する
4. **構造化:** `src/` レイアウトを採用し、ビジネスロジックと設定ファイルを分離する

## 3. コード生成・実行時のルール
- **ライブラリ追加:** パッケージのインストールが必要な場合は、`uv add <package>` コマンドを提示する。
- **スクリプト実行:** `python <file>` ではなく、仮想環境を自動認識する `uv run <file>` を推奨する。
- **ツール実行:** ruffやmypy等の静的解析ツールは `uvx <tool>` または `uv run <tool>` で実行する手順を示す。

## 4. WSL/VS Code 連携設定
- `.vscode/settings.json` にて、`python.defaultInterpreterPath` を `${workspaceFolder}/.venv/bin/python` に設定するガイドを含める。

#コード生成する際は、以下の点に注意してコード生成して下さい。
. **Security & Integrity (セキュリティと整合性)**
   - 一般的な脆弱性（インジェクション、XSS、認証不備）はないか？
   - エラーハンドリングは適切か？（`try-catch`で握りつぶしていないか？ログは出力されているか？）
   - ハードコーディングされた機密情報（APIキー、パスワード）やマジックナンバーはないか？
   - 存在するかわからない怪しい関数（ハルシネーション）を呼んでいないか？

2. **Architecture & Performance (構造と性能)**
   - 冗長（Redundant）なロジックや、車輪の再発明をしていないか？（標準ライブラリで代替可能か？）
   - 「すべての状況」を想定しているか？（Edge caseの考慮漏れはないか？）
   - 非推奨（Deprecated）なメソッドや古いライブラリを使用していないか？
   - 責務の分離は適切か？（スパゲッティコードになっていないか？）

3. **Maintainability (保守性)**
   - 変数名・関数名は具体的で説明的か？（`data`, `process`, `temp` などの曖昧な命名は禁止）
   - コメントはコードの動作を翻訳しているだけではないか？（「なぜそうしたか」の意図が書かれているか？）
   - コンテキストの不整合（冒頭と末尾で設計思想がブレている）はないか？
